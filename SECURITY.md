# Security Policy

## Supported Packages

Security fixes will target the current release line of:

- `@cmd-kit/core`
- `@cmd-kit/react`

`apps/web` is not a published npm package, but security-sensitive issues affecting the docs or playground should still be reported.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

Until a dedicated security contact address exists, report vulnerabilities privately to the project maintainer through the repository host's private reporting channel or direct maintainer contact.

Include:

- affected package and version
- impact summary
- reproduction details
- any suggested mitigation

## Response Expectations

- initial acknowledgement target: within 7 days
- remediation plan target: as soon as impact is confirmed
- coordinated disclosure preferred until a fix is available
